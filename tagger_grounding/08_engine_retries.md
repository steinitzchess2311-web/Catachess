# Stage 08 - 引擎调用与重试策略

目标：保证 engine 调用稳定，不拖垮 worker。

## 一、调用参数
- engine URL: https://sf.catachess.com/engine
- depth / multipv 可配置

## 二、超时与重试
- 单次请求超时：例如 10-20s
- 重试次数：最多 3 次
- 退避策略：指数退避（1s / 2s / 4s）
 - 记录每次重试的错误码与耗时

## 三、失败降级
- 单盘失败不影响整体上传
- 失败盘记录原因（超时/503/解析错误）

## Checklist
- [ ] 超时与重试上限定义
- [ ] 退避策略确认
- [ ] 失败记录字段确认
